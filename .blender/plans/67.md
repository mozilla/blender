# Plan: Link auto-merge comment to the workflow run

## Summary
When BLEnder auto-merges a Dependabot PR, it leaves an approval review whose body reads
`BLEnder auto-merge: all safety gates passed (CI green, patch/minor, compat 82%, no advisories).`
Unlike the fix, review-major, and alert-investigation comments — which all append a
`[Workflow run](<url>)` markdown link — this approval body has no link back to the run that
produced it. The fix is to build the run URL from the standard GitHub Actions environment
variables (`GITHUB_SERVER_URL`, `GITHUB_REPOSITORY`, `GITHUB_RUN_ID`) exactly as
`post_alert_action.py._run_url()` already does, and append the link to the approval review
body (and, for consistency, to the skip/will-not-merge comments). No workflow YAML change is
needed because those three variables are injected automatically into every Actions step.

## Files to Change

- `scripts/automerge_dependabot.py` —
  - Add a small `_run_url() -> str` helper (mirroring `post_alert_action.py:162-169`) that
    returns `f"{server}/{repository}/actions/runs/{run_id}"` when all three env vars are
    present, else `""`.
  - In `approve_and_merge()`, append a trailing ` [Workflow run](<url>)` to `review_body`
    when `_run_url()` is non-empty (gracefully omit the link when it is empty, e.g. local
    runs/tests). Keep the existing sentence intact so dedup/prefix matching on
    `"BLEnder auto-merge:"` is unaffected.
  - (Consistency, recommended) In `_post_skip_comment()`, append the same
    ` [Workflow run](<url>)` suffix to both the retry and will-not-merge comment bodies.
    Note the dedup check matches on `c.body.startswith("BLEnder: ")`, which the suffix does
    not break.

- `tests/scripts/test_automerge_dependabot.py` — add unit tests (see Test Strategy).

## Implementation Steps

1. Add the `_run_url()` helper near the other module-level helpers (e.g. just above
   `approve_and_merge`) in `scripts/automerge_dependabot.py`. Use `os.environ.get(...)` with
   `""` defaults; `os` is already imported.
2. Update `approve_and_merge()` to compute `run_url = _run_url()` and build
   `review_body = base_sentence + (f" [Workflow run]({run_url})" if run_url else "")`.
3. (Recommended) Update `_post_skip_comment()` to append the same suffix to both comment
   variants, computed once from `_run_url()`.
4. Optionally factor the helper into `scripts/github_utils.py` (e.g. `run_url()`) and have
   both `automerge_dependabot.py` and `post_alert_action.py` import it to remove duplication —
   flag as optional cleanup to avoid widening scope.
5. Run the full pre-push check suite (ruff check/format, pytest) and the lint jobs listed in
   `.blender/agents.md`.

## Test Strategy

- New test: with `GITHUB_SERVER_URL=https://github.com`, `GITHUB_REPOSITORY=owner/repo`,
  `GITHUB_RUN_ID=12345` set via `monkeypatch.setenv`, call `approve_and_merge(mock_pr, 82)`
  with a `MagicMock` PR; assert `mock_pr.create_review` was called with `event="APPROVE"` and
  a body that both starts with `"BLEnder auto-merge: all safety gates passed"` and contains
  `"[Workflow run](https://github.com/owner/repo/actions/runs/12345)"`. Patch
  `enable_auto_merge` (or import the real one with a mocked requester) so the call is inert.
- Fallback test: with the three env vars deleted (`monkeypatch.delenv(..., raising=False)`),
  assert the review body contains the base sentence and does NOT contain `"[Workflow run]"`.
- Compat-unknown test: pass `compat_score=None` and assert body shows `compat unknown` plus
  the link, confirming the link is independent of score formatting.
- (If skip comments are updated) extend an existing `_post_skip_comment` path test to assert
  the run link is appended and that the `startswith("BLEnder: ")` dedup still matches.
- Run `uv run --extra test pytest tests/` plus `ruff check scripts/` and
  `ruff format --check scripts/`.

## Risks

- **Env var availability:** `GITHUB_SERVER_URL`/`GITHUB_REPOSITORY`/`GITHUB_RUN_ID` are
  Actions defaults, so the `chore-automerge-dependabot-prs.yml` "Run auto-merge" step needs no
  edit; but if a future runner context lacks them the link is silently omitted (acceptable,
  matches `post_alert_action.py` behavior). Verify in a real run that the variables are
  populated.
- **Dedup/prefix matching:** Several checks rely on body prefixes (`"BLEnder: "`,
  `"BLEnder auto-merge:"`). Appending the link as a suffix keeps prefixes intact, but any
  future exact-equality comparison on these bodies would break — keep matches prefix-based.
- **Review vs. issue comment:** The auto-merge message is a PR *review* body, not an issue
  comment; GitHub renders markdown links in review bodies, so the link will display correctly.
- **Scope creep:** Centralizing the helper into `github_utils.py` touches
  `post_alert_action.py`; keep that optional to limit blast radius unless reviewers prefer the
  dedup.
